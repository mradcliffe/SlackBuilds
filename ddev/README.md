<p><a href="https://circleci.com/gh/drud/ddev" rel="nofollow"><img src="https://camo.githubusercontent.com/524a2f09fc6cf540096425626f0639a11453c119/68747470733a2f2f636972636c6563692e636f6d2f67682f647275642f646465762e7376673f7374796c653d736869656c64" alt="CircleCI" data-canonical-src="https://circleci.com/gh/drud/ddev.svg?style=shield" style="max-width:100%;"></a> <a href="https://goreportcard.com/report/github.com/drud/ddev" rel="nofollow"><img src="https://camo.githubusercontent.com/fceffce9d1f39ae1a97bc1571fba6364ffdc7dae/68747470733a2f2f676f7265706f7274636172642e636f6d2f62616467652f6769746875622e636f6d2f647275642f64646576" alt="Go Report Card" data-canonical-src="https://goreportcard.com/badge/github.com/drud/ddev" style="max-width:100%;"></a> <a target="_blank" rel="noopener noreferrer" href="https://camo.githubusercontent.com/0a09e1fce949a9b283ff4590c3cd501e29596594/68747470733a2f2f696d672e736869656c64732e696f2f6d61696e74656e616e63652f7965732f323032302e737667"><img src="https://camo.githubusercontent.com/0a09e1fce949a9b283ff4590c3cd501e29596594/68747470733a2f2f696d672e736869656c64732e696f2f6d61696e74656e616e63652f7965732f323032302e737667" alt="project is maintained" data-canonical-src="https://img.shields.io/maintenance/yes/2020.svg" style="max-width:100%;"></a></p>
<p><a target="_blank" rel="noopener noreferrer" href="/drud/ddev/blob/v1.15.3/images/ddev_logo.png"><img src="/drud/ddev/raw/v1.15.3/images/ddev_logo.png" alt="ddev logo" style="max-width:100%;"></a></p>
<p>ddev is an open source tool that makes it simple to get local PHP development environments up and running in minutes. It's powerful and flexible as a result of its per-project environment configurations, which can be extended, version controlled, and shared. In short, ddev aims to allow development teams to use Docker in their workflow without the complexities of bespoke configuration.</p>
<h2><a id="user-content-getting-started" class="anchor" aria-hidden="true" href="#getting-started"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Getting Started</h2>
<ol>
<li><strong>Check System Requirements:</strong> We support recent versions of macOS, Windows 10, and Linux distributions that will run Docker (ddev requires Docker and docker-compose). (<a href="https://ddev.readthedocs.io/en/stable/#system-requirements" rel="nofollow">more info here</a>).</li>
<li><strong>Install ddev:</strong> <a href="https://ddev.readthedocs.io/en/stable/#installation" rel="nofollow">Options include</a> macOS homebrew (recommended), an install script, or manual installation.</li>
<li><strong>Read the Basics and Choose a CMS Quick Start Guide:</strong>

<h2><a id="user-content-partial-feature-list" class="anchor" aria-hidden="true" href="#partial-feature-list"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Partial Feature List</h2>
<ul>
<li>Quickly create multiple local web development environments based on code repositories.</li>
<li>Import database for a project you're working on.</li>
<li>Import upload files to match the project (e.g. Drupal's sites/default/files or WordPress's wp-content/uploads).</li>
<li>Pantheon integration - grab a Pantheon archive and work locally with the database and files.</li>
<li>Run commands within the docker environment using <code>ddev exec</code>.</li>
<li>View logs from the web and db containers.</li>
<li>Use <code>ddev ssh</code> to explore the linux environment inside the container.</li>
<li>List running projects with <code>ddev list</code>.</li>
<li>Snapshot databases with <code>ddev snapshot</code>.</li>
<li>Temporarily share your development website with other using <code>ddev share</code>.</li>
<li>Trusted HTTPS support</li>
</ul>
<p>Just running <code>ddev</code> will show you all the commands.</p>
